The SSMP Packet

The SSMP Packet is made up of three main sections:

  1. The main header
  2. The user header
  3. The payload

Both the user-defined header and the payload are optional and mutually exclusive. All packets contain the main header, but only some packets have a user header and/or a payload.

If all three are present, the user header follows the main header, and the payload follows the user header. If there's no user header, the payload is directly after the main header.

The payload MUST be the last section of the packet, since the main header doesn't explicitly define a 'payload length' field.

A SSMP packet with a payload and user header would look like this:

//       +----------------+----------------+----------------+----------------+
// Byte  |       0        |       1        |       2        |       3        |
//       +================+================+================+================+
//  +0   |     Magic      |    Version     |  Control Byte  |    Checksum    |
//       +----------------+----------------+----------------+----------------+
//  +4   |                        Total Packet Length                        |
//       +-------------------------------------------------------------------+
//  +8   |                           Payload Offset                          |
//       +-------------------------------------------------------------------+
//  +12  |                            User Header                            |
//  ...  |                                ...                                |
//       +-------------------------------------------------------------------+
//  +N   |                              Payload                              |
//  ...  |                                ...                                |
//       +-------------------------------------------------------------------+

Checksums

Each packet has a checksum, stored in the main header. Presently, checksums are calculated by adding every byte in the packet (including payload and user-defined header bytes, and excluding the checksum byte itself) and representing the result as a u8. Integer overflow is allowed while computing the checksum.

Checksums are verified by subtracting every byte (excluding the checksum byte) from the checksum stored in the header. If the result is 0, the sum is verified. Otherwise, it's not; the packet has been tampered with.
